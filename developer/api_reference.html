pvfactors.pvgeometry¶

In pvfactors, we need a way to store efficiently shapely geometries in Dataframes and perform only basic geometric operations. So in this case, using a complete package like geopandas may be overkill since it brings with it a lot of dependencies, and may complicate the integration of the package in other open-source projects. Here we implement the basic functionalities of geopandas that are useful in pvfactors.

Classes¶

pvgeometry.PVGeometry Lightweight extension of pandas.DataFrame that copies functionalities of geopandas.GeoSeries

Methods and properties¶

PVGeometry.split_ground_geometry_from_edge_points Break up ground lines into multiple ones at the pv row “edge points”, which are the intersections of pv row lines and ground lines.
PVGeometry.break_and_add_entries Break up a surface geometry into two objects at a point location.

PVGeometry.split_pvrow_geometry Break up pv row line into two pv row lines, a shaded one and an unshaded one.

All the functions necessary for timeseries calculations.

Functions¶

array_timeseries_calculate Calculate the view factor radiosity and irradiance terms for multiple times.
perez_diffuse_luminance Function used to calculate the luminance and the view factor terms from the Perez diffuse light transposition model, as implemented in the pvlib-python library.

add_df_registries_nans df_registries has nan entries for timestamps that were skipped, make sure to add them back in with this function
